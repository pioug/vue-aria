# @vue-aria/i18n

Locale and direction primitives.

## `provideI18n`

Provides locale and text direction to child hooks/components.

## `useLocale`

Returns the current locale and direction (`ltr` or `rtl`).

## `useDefaultLocale`

Returns detected locale/direction when no provider is present.

## `useLocalizedStringDictionary`

Returns a cached localized dictionary for the current package/message map.

## `useLocalizedStringFormatter`

Returns a locale-aware formatter for package message dictionaries.

## `isRTL`

Utility for determining direction from a locale string.

```ts
import {
  provideI18n,
  useLocale,
  useLocalizedStringDictionary,
  useLocalizedStringFormatter,
  isRTL,
} from "@vue-aria/i18n";

provideI18n({ locale: "ar-EG" });
const locale = useLocale();
const dictionary = useLocalizedStringDictionary({
  "en-US": { pending: "pending" },
});
const formatter = useLocalizedStringFormatter({
  "en-US": { pending: "pending" },
});

isRTL("he-IL"); // true
formatter.value.format("pending");
```

### Behavior

- Inherits locale values through nested providers.
- Supports direction override independent of locale.
- Defaults to browser locale when no provider is configured.
